Note: The source file uses Unix LF (0x0A) line endings.
IBM-1047 maps ASCII LF (0x0A) to EBCDIC 0x25.
HTTPD must translate EBCDIC 0x25 back to ASCII LF for the browser.
CR+LF (0x0D 0x0A) should also work: CR maps to 0x0D, LF to 0x25.
